# 阶段二自主任务路线

## 隔离基线

阶段一稳定版本固定为Git标签：

```text
stage1-stable-v1.0.0
```

阶段二开发分支：

```text
codex/stage2-autonomy
```

阶段二不修改机器人动作文件和`UART1TOUART2`透明桥固件。阶段一和阶段二必须使用不同systemd服务，且不能同时访问摄像头和串口。

## 目标状态机

```text
IDLE
-> SEARCH_PILLAR
-> APPROACH_PILLAR
-> ORBIT_PILLAR
-> ALIGN_CARD
-> READ_TASK
-> SEARCH_GONG
-> APPROACH_GONG
-> EXECUTE_ACTION
-> DONE
```

任何状态丢失关键目标或运行超时，都必须先停止底盘，再恢复搜索或进入`FAILSAFE`。

## 模型类别

初始检测模型计划使用三个类别：

```text
pillar
task_card
gong
```

任务1和任务2不作为YOLO类别区分。YOLO只定位任务纸，随后裁剪任务纸区域并交给现有RapidOCR读取文字。

## 实施里程碑

1. 冻结阶段一并建立独立分支。
2. 完成状态机和离线测试，不连接机器人。
3. 收集柱子、任务纸、锣和场地背景数据。
4. 在电脑上标注并训练轻量YOLO模型。
5. 导出ONNX并转换为Ascend310B4的OM模型。
6. Atlas摄像头实时检测，只显示结果，不发送运动命令。
7. 录像回放状态机，只记录拟发送命令。
8. 架空底盘进行低速运动测试。
9. 实现场地内接近、绕柱、OCR、寻找锣和动作闭环。
10. 创建独立但默认禁用的阶段二systemd服务。

## 开始训练前需要的材料

- 两根柱子各个方向、不同距离的照片或视频。
- 锣在场地中的正面、侧面、远距离和遮挡照片。
- 任务纸贴在柱子上的实际尺寸和打印样式。
- 场地俯视图、尺寸、机器人起点和障碍物位置。
- 是否必须访问两根柱子，还是发现任意一张任务纸即可。
- 比赛限时和允许使用的附加传感器。
