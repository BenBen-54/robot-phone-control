# 手机网页控制下一步

这份文档从“`tools/zys_udp_test.py` 已经能让机器人按预期移动”开始。

## 1. 网络连接方式

保持现在这套连接：

- 机器人开机并开启 `MYAP-XXXXXX` 热点。
- 电脑连接机器人 `MYAP-XXXXXX` 热点。
- 手机也连接同一个 `MYAP-XXXXXX` 热点。
- 官方 APP、动作编辑器、命令行测试脚本都先关闭。

默认机器人地址：

```text
192.168.4.1
```

## 2. 在电脑启动网页控制服务

PowerShell 进入项目目录：

```powershell
cd C:\Users\20914\Documents\robot-phone-control
```

启动真实 UDP 控制模式：

```powershell
python server\main.py --host 0.0.0.0 --port 8000 --robot-mode udp --robot-ip 192.168.4.1 --robot-password 88888888
```

这个窗口不要关。后续手机按钮会通过这个 Python 服务转发 UDP 指令给机器人。

## 3. 找到电脑在机器人热点里的 IP

再开一个 PowerShell：

```powershell
ipconfig
```

找到无线网卡的 IPv4 地址。常见是：

```text
192.168.4.2
```

也可能是：

```text
192.168.4.3
```

## 4. 手机打开控制台

手机连上同一个 `MYAP-XXXXXX` 后，在手机浏览器打开：

```text
http://电脑IPv4地址:8000
```

例如：

```text
http://192.168.4.2:8000
```

看到“智元素控制台”后，状态显示“已连接”就可以测试。

## 5. 推荐测试顺序

先把机器人轮子架空。

1. 点击“电池”，确认后端能收到按钮命令。
2. 点击“模式0”。
3. 把速度保持在 10。
4. 按住“前进”，松手后会自动停止。
5. 依次测试后退、左移、右移。
6. 再测试左转、右转。
7. 最后测试“普攻”或“开机动作”。

如果机器人状态异常，优先点击红色“停止”。

## 6. 常见问题

如果手机打不开网页：

- 确认手机和电脑都连接同一个 `MYAP-XXXXXX`。
- 确认启动命令里是 `--host 0.0.0.0`。
- 确认手机访问的是电脑的 `192.168.4.x` 地址，不是机器人 `192.168.4.1`。
- Windows 防火墙弹窗时允许 Python 在专用网络通信。

如果网页连接成功但机器人不动：

- 先回到命令行运行 `python tools\zys_udp_test.py --robot-ip 192.168.4.1 move --angle 0 --speed 10 --turn 0 --duration 0.25` 确认底盘仍然可控。
- 关闭官方 APP 和动作编辑器。
- 重启网页控制服务。

如果提示 UDP 9999 端口被占用：

- 关闭官方 APP、动作编辑器、之前运行的测试脚本。
- 关闭已经启动的 `python server\main.py` 窗口。
- 重新启动网页控制服务。
